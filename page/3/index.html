CG DCG NDCG 系列

事实上，该系列用于最初用于评价搜索引擎的好坏。全部以 @K 为例。

CG

只考虑每个结果的分数信息。如果是评分预测，则是前 K 个的真实分数的加和。如果是 0/1 的形式，则实际上就是前 K 个中预测到的个数。

CGp=∑i=1preli\mathrm{CG}_{\mathrm{p}}=\sum_{i=1}^{p} r e l_{i}CGp​=i=1∑p​reli​

DCG

评分预测中，评分高的应该越靠前。0/1 的预测中，实际上真的出现的应该更靠前。

DCGp=rel1+∑i=2prelilog⁡2i\mathrm{DCG}_{\mathrm{p}}=r e l_{1}+\sum_{i=2}^{p} \frac{r e l_{i}}{\log _{2} i}DCGp​=rel1​+i=2∑p​log2​ireli​​

或是

DCGp=∑i=1p2reli−1log⁡2(1+i)\mathrm{DCG}_{\mathrm{p}}=\sum_{i=1}^{p} \frac{2^{r e l_{i}}-1}{\log _{2}(1+i)}DCGp​=i=1∑p​log2​(1+i)2reli​−1​

NDCG

不同 query 的搜索结果有多有少，所以不同 query 的 DCG 值就没有办法来做对比。
所以就用预测到的 DCG 除以理想情况下的 DCG（记为IDCG）

NDCGu@k=DCGu@kIDCGuN D C G_{u} @ k=\frac{D C G_{u} @ k}{I D C G_{u}}NDCGu​@k=IDCGu​DCGu​@k​

Tip: 每个 user 的应该单独计算，最后求均值

Hit Ratio @ K

一种常用的衡量召回率的指标。查到了两种定义方式：

  1. 分母是所有的测试集合，分子是每个用户前K个中属于测试集合的个数的总和

HR@K=Number OfHits@KGTH R @ K=\frac{\text {Number OfHits@K}}{G T}HR@K=GTNumber OfHits@K​

  1. 对于相似区域搜索问题：每一个查询的区域，会生成（或指定）一个正样本。对于查询的 K 个返回结果，如果其中包含该正样本，则 HR@K = 1，否则 HR@K = 0

MRR

衡量正确检索结果值在检索结果中的排名。

MRR=1Q∑i=1∣Q∣1rankiM R R=\frac{1}{Q} \sum_{i=1}^{|Q|} \frac{1}{r a n k_{i}}MRR=Q1​i=1∑∣Q∣​ranki​1​

其中，∣Q∣是用户的个数，ranki 是对于第 i 个用户，推荐列表中第一个在ground-truth 结果中的 item 所在的排列位置。

Accuracy

准确率表示预测正确（正负均正确）的样本数占总样本书的比例。

Accuracy=TP+TNTP+TN+FP+FN\text {Accuracy}=\frac{T P+T N}{T P+T N+F P+F N}Accuracy=TP+TN+FP+FNTP+TN​

Precision

精确率表示预测为正样本的样本中，正确预测为正样本的概率。

Precision=TPTP+FPP r e c i s i o n=\frac{T P}{T P+F P}Precision=TP+FPTP​

Recall

在分类任务中，召回率表示正确预测出正样本占实际正样本的概率。

Recall=TPTP+FN\text {Recall}=\frac{T P}{T P+F N}Recall=TP+FNTP​

在推荐系统中，可以看作是 预测到与 user 会有交互的 item 数占所有与该用户有交互的 item 的比例。

 Recall =∑u∈U∣R(u)∩T(u)∣∣T(u)∣\text { Recall }=\frac{\sum u \in U|R(u) \cap T(u)|}{|T(u)|} Recall =∣T(u)∣∑u∈U∣R(u)∩T(u)∣​

Recall@K 则指的是 R(u) 只在前 K 个中找。

Spatiooral Dual Graph Attention Network for Query-POI Matching

Posted on 2020-10-23

文章主要解决什么问题

POI 的查询与匹配在各类 location-based 的服务中愈发重要。例如导航软件、打车软件。现在的应用场景需要根据用户输入的不完整的信息，来尽量准确的推荐 POI。

此外，除了应对普通的查询请求，更多的查询需要考虑不同的用户、不同的地点、不用的语境和时间关系，做出更精准的匹配。

文章可以应对三种困难：

  • 不确定性：同名 POI 可能有好几个地点
  • 复杂的当前状态: 例如需要考虑距离、当前时间等因素
  • 动态的用户兴趣: 用户的兴趣会发生变化，但和时间序列有关

解决的思路是什么

  • 使用一个语义提取模块，通过 CNN 等各种方法，提取搜索文本、POI名、时间、坐标、用户的特征信息，转为低维向量
  • 使用两个 Graph Attention Network，获取 POI 与 查询的相关性。第一个图用于获取通用的、相对地理位置有关的 query-POI 相关性，第二个图用于获取特定用户的、事件相关的 query-POI 相关性
  • 利用一个 fully-connected 网络来融合两个图获取到的信息
  • 利用 cos 或类似的相关性计算函数来获得 query-POI 的相关性

核心的知识点有哪些

  • Embedding
  • Graph Attention Network

模型分块说明

semantic representation

利用共享参数的 CNN 将 POI-name 和 query 编码为低维名称向量表示
按经纬度划分网格，将当前经纬度所在的网格和周围网格，使用 CNN 联合编码为当前位置对应的低维位置向量。再使用 CNN 来获取每一个查询对的低维向量表示。
将时间按一定间隔划分为时段。将当前时刻映射到对应时间段内，使用该时间段的向量表示
将每个用户的交互历史按照时段划分，将 item 按照对应的种类归类，得到用户在某个时间段内与某一类 item 交互的比例。然后使用 fully-connected 编码为低维的用户向量表示

dual graph attention network

generic query-POI graph

当某个 query 和某个 POI 之间的查询事件次数超过阈值，就连上一条权值为 1 的边。每个点的表示为上个模块中提取的语义信息. 使用 graph attention network 来获取普适的 query / POI 的 embedding。其中的 attention 为：

user-specific query-POI graph

此处以 POI 为例，对于 query 的处理类似。

总之，虽然有一些我暂时不能理解的卷积和 GCN 操作，但是大体的感觉是，通过邻居的 embedding 来丰富自身表示；利用上一个时刻的信息来影响下一时刻的信息。

pairwise neural fusion and ranking & output

将两个 graph 得到的 embedding 做一个 concatenation 操作拼接到一起，放入一个 fully-connected network 中得到 query 和 POI 最终的 embedding 表示。
最后使用 cos 相似度计算方法来计算相关性。

存在的问题 & 有哪些改进的思路

我个人觉得，把 POI 的 embedding 和 query 的 embedding 训练的一样并不太 make sense. 可能还是像 TransR / TransE 那样或许合理一点。那么时间信息和用户信息可以用来增强 edge 的信息。

Spatial Object Recommendation with Hints: When Spatial Granularity Matters

Posted on 2020-10-20 Edited on 2020-10-21

文章主要解决什么问题

灵活的多级/多层 POI 推荐。
可解释的 POI 推荐。

解决的思路是什么

将 POI 之间的相互包含关系预处理为一棵 POI 树。
为 POI 的每层都安排一个任务，利用 user-POI 交互与 POI 树结构，给每层做 top-k 推荐。
根据各种信息在最终得分中的贡献做推荐(用户层面、POI层面、交互层面)

核心的知识点有哪些

  • multi-task
  • feature engineering

模型分块说明

  • multi-task
  • subtaskA Atrribute-based Representing Learning
    这一部分是利用 user 和 POI 自有的信息做特征提取。一方面是完全利用自身的信息构造直接特征矩阵，如 POI 的类型，user 的年龄等等。另一方面，利用了 POI 历史用户信息来表示 user 的一些特征，即反序特征矩阵；利用 user 表示 POI 亦然。【这里指的是统计层面的数据，没有利用到具体的 user POI 交互】
    最终利用矩阵分解得到 attributed-based 的特征向量。
  • subtaskB Interaction-based Representing Learning
    利用 co-search co-visit distance 这三个关系得到空间联系信息，乘上 attributed 的 user 矩阵，得到基于空间的用户 rating-matrix。
    利用 POI 之间每一层的关系来丰富 POI 表示。即利用下层来丰富上层的表示。利用 POI 的这个带有层次信息的特征矩阵，乘上 user 矩阵，得到基于特征的用户 rating-matrix。
    组合两个 ratting-matrix 得到最终的矩阵，使用 BPR 训练各参数。

存在的问题

  • POI tree 是预定义的。
  • 只有上层利用下层的结果，所以最下层的效果可能不好。

Efficient similar region search with deep metric learning

Posted on 2020-10-17 In UrbanComputing

文章主要解决什么问题

在区域搜索中，存在两个问题：

  1. 相似度衡量需要更合理的方法
  2. 搜索的时间复杂度极高

解决的思路是什么

  1. 针对相似度衡量问题，定义了一种可以既可以衡量区域内部属性，又可以衡量区域内 item 的相对位置。
  2. 针对搜索时间复杂度较高的问题，使用了一种特征预处理及分支限界法来解决。

核心的知识点有哪些

  1. 三元 CNN 网络 -> 用于比较排名
  2. 基于分支限界思想的 Efficient subwindow search 的搜索算法

模型分块说明

  1. 排名学习模型（用于衡量相对相似性）
    1. 三元组网络
      三元组 CNN 网络在图像识别领域用的比较多，应用于图像的排序度量。像正常的 CNN 一样使用各种 kernal 来捕获局部特征。CNN 的最后一层输出的特征（不是全连接层之后得到的一个标量）内容被记为 Net(*)。在解决排名问题时这样训练：任意挑选一个查询样本 xq ，找出一个与其类别相同的正样本 x+ 以及一个与其类别不同的负样本 x-。
      那么正负样本与查询样本之间的距离即为

      d+=∥Net⁡(xq)−Net⁡(x+)∥2 and d−=∥Net⁡(xq)−Net⁡(x−)∥2d_{+}=\left\|\operatorname{Net}\left(x_{q}\right)-\operatorname{Net}\left(x_{+}\right)\right\|_{2} \text { and } d_{-}=\left\|\operatorname{Net}\left(x_{q}\right)-\operatorname{Net}\left(x_{-}\right)\right\|_{2}d+​=∥Net(xq​)−Net(x+​)∥2​ and d−​=∥Net(xq​)−Net(x−​)∥2​

      显然，如果我们想要这个模型能够正确排名，就需要 d+ 尽可能小，d- 尽可能大。所以就有了 loss function 为：

      L=∑(xq,x+,x−)max⁡{0,d+−d−+δ}+λ∥Net⁡(⋅)∥2\mathcal{L}=\sum_{\left(x_{q}, x_{+}, x_{-}\right)} \max \left\{0, d_{+}-d_{-}+\delta\right\}+\lambda\|\operatorname{Net}(\cdot)\|_{2}L=(xq​,x+​,x−​)∑​max{0,d+​−d−​+δ}+λ∥Net(⋅)∥2​

    2. 本论文中的使用方法
      对于本文的区域问题，首先将区域划分成一个个小格子（类比像素），区域特征 R∈Rw×h×dR \in \mathbb{R}^{w \times h \times d}R∈Rw×h×d 经过 CNN Net 之后提取到的信息为 X∈Rw∗×h∗×KX \in \mathbb{R}^{w^{*} \times h^{*} \times K}X∈Rw∗×h∗×K，然后再使用一个聚合函数 g(*) （此处为 max-pooling）获取这个区域信息的一维表示 v=g(X),v∈R1×K\mathbf{v}=g(X), \mathbf{v} \in \mathbb{R}^{1 \times K}v=g(X),v∈R1×K。所以本文中对应的 d+ 和 d- 为：d+=∥vq−v+∥2d_{+}=\left\|\mathbf{v}_{q}-\mathbf{v}_{+}\right\|_{2}d+​=∥vq​−v+​∥2​ and d−=∥vq−v−∥2d_{-}=\left\|\mathbf{v}_{q}-\mathbf{v}_{-}\right\|_{2}d−​=∥vq​−v−​∥2​。这样就可以进行 CNN Net 的训练了
      训练完之后，区域相似度可以表示为：sim⁡(R1,R2)=11+∥v1−v2∥2\operatorname{sim}\left(R_{1}, R_{2}\right)=\frac{1}{1+\left\|\mathbf{v}_{1}-\mathbf{v}_{2}\right\|_{2}}sim(R1​,R2​)=1+∥v1​−v2​∥2​1​
    3. 训练 robust 的三元组网络
      由于没有标好标记的区域相似性数据，文中提出一种自监督的学习策略，通过学习自相似性来使模型具备 robust，无需再获取标签。
      所谓的 robust 可以理解为，一个原始区域和一个由前者经少量噪声和移动而产生的区域应被视为是相似的。
      按照这个思路，随机挑选一个区域作为查询区域，加上少量的噪声和移动作为正样本；随机选取其他区域作为负样本、原查询区域加上过多的噪声和移动作为难例负样本。
      这样，模型学到的特征对小的噪声和移动也就具有了 robust。
    4. 基于比率的训练损失
      不同区域中的目标数差别很大，这可能会导致一些训练数据组的损失值很大，盖过了其它训练数据的影响。所以修改 loss function 为：

      L=∑(Rq,R+,R−)max⁡{0,d+d++d−−δ}+λ∥Net⁡(⋅)∥2\mathcal{L}=\sum_{\left(R_{q}, R_{+}, R_{-}\right)} \max \left\{0, \frac{d_{+}}{d_{+}+d_{-}}-\delta\right\}+\lambda\|\operatorname{Net}(\cdot)\|_{2}L=(Rq​,R+​,R−​)∑​max{0,d+​+d−​d+​​−δ}+λ∥Net(⋅)∥2​

  2. 区域搜寻算法（解决搜索时间极长的问题）

    1. 通过共享 CNN 计算结果转化问题
      计算相似性的时候，每选一个候选区域，就要通过 CNN 来计算它的特征。如果要查询很多次，特别是有部分区域重叠的时候，就会有许多重复计算。那干脆直接预处理最大的区域，然后直接找特征相似的区域。找到特征相似区域之后，再反向找出原区域即可。

    2. 分支限界法求精准结果

文章主要解决什么问题

空间知识图谱中的一种常见应用（也是一种常见问题）是相似区域搜索。相似区域搜索需要解决两个关键的问题：

  • 如何对空间知识图谱中的 entity 做合理的 embedding，以提取有用的信息
  • 如何衡量衡量区域相似度
    本文解决了上述两个问题

解决的思路是什么

  • 利用 entity 的类别层次信息，即类别树的关系，合理进行 embedding
  • 使用 CNN 来融合一个区域内的所有 entity 信息，因为 CNN 可以获取区域特征，变相保留了相对位置信息

核心的知识点有哪些

  • embedding
  • feature engineering
  • CNN

模型分块说明

  1. 如何对 entity 进行合理的 embedding
    embedding 的原则是：具有相似层次类型的 / 语义相似的 entity 对应该越相近。现有的知识图谱 embedding 方法基于各种 Trans，可能不能很好的提取类别关系（如大类别包含小类别等关系）。本文作者提出：借助 entity 的类型层次和语义相似度，定义距离，用于 embedding.

    • 层次类型信息
      思想是：所有类别中，与该类别有关的类别数量越少，该类别所包含的信息量越大（信息熵）。两个 entity 在类别上的层次相似度，可以看作是规格化的最近公共类别节点的信息量。

      Cht=γ×(1−csim⁡(h,t)max⁡IC(⋅))C_{h t}=\gamma \times\left(1-\frac{c \operatorname{sim}(h, t)}{\max I C(\cdot)}\right)Cht​=γ×(1−maxIC(⋅)csim(h,t)​)

      csim⁡(h,t)=max⁡IC(c)c \operatorname{sim}(h, t)=\begin{aligned}\max & I C(c)\end{aligned}csim(h,t)=max​IC(c)​

      IC(c)=−log⁡p(c)I C(c)=-\log p(c)IC(c)=−logp(c)

      其中 p© 表示访问所有层次的类别时，遇到类别 c 的概率。显然这个概率越大，类别 c 所包含的信息越少。
    • 语义相似度信息
      根据两个 entity 所对应的类别在类结构图上所相距的跳数，获得语义信息。具体的跳数越远，相似度越小，反之越大。

    传统方法可能是将一个区域内所有 entity 的特征叠加，但是这样就失去了语义信息。作者的思路是，模仿计算机图像处理中，使用 CNN 来提取区域特征。这样，就可以在利用各个 entity 特征的同时，融合他们的相对位置信息。

leetcode-18-四数之和

Posted on 2020-10-05 In leetcode

最朴素的做法是 O(n^4) 的暴力算法，显然不行。

第一个优化的思路，前三个数循环，然后最后一个数 (target - 前三个数) 用 set 判断是否存在。复杂度是 O(n^3 * logn)

第二个优化思路。前两个数循环，然后用两个 L R 指针分别指向 j+1 和 n-1。根据四个数和的大小左右移动即可。这样复杂度是 O(n^3)。

左右指针是一个常见的用法，特此记录。

Exploring the Urban Region-of-Interest through the Analysis of Online Map Search Queries

Posted on 2020-10-04 In UrbanComputing

文章主要解决什么问题

识别 Region of Interest 并分析所设计的各类 topic。
ROI 与 POI 不同，他包含了多种 POI ，有多种 function / categories。往大了做，可以以一个商圈的粗粒度来使用，往小了做，可以看成是融合了几种类型的综合店铺。

解决的思路是什么

  • 利用 density-based 聚类方法来识别 ROI
  • 利用 topic model 分析查询历史，挖掘更多 ROI 的信息

核心的知识点有哪些

  • clustering
  • topic model

模型分块说明

  • clustering
    将城区划分为细粒度的网格，利用 PageRank 分析各个网格的受欢迎程度。再利用 Heat Value 等指标对网格进行聚类融合，得到不同的 ROI。
  • topic model
    NLP 相关，此处跳过。

其他

这篇论文的数据分析部分很值得一看。

  • topic model
    以北京市为例，通过 topic 的空间分布(user 被 topic 所属地点吸引)、topic 所包含的 POI 类别、topic 是时间分布这三个指标，可以分析出一些场景。如：去工作、家庭晚宴、周末购物等场景，可以为城市商业规划提供指导。
  • Topic-Aware Visitor Inference for ROIs
    识别 topic 相关的 ROI 访问者来源。例如国贸的访客有好几种 topic 组成，每种 topic 对应的访客的到访目的都不一样。

    也可以和地域分析结合在一起。餐饮购物相关的 topic 从北京东部吸引的客流较少，因为北京东部也有许多购物中心。同时，对于海淀区的游客来说，Topic 7 没有 Topic 15那么有吸引力，因为那里的人更喜欢五道口的餐馆。

这种不同的 topic 吸引了不同的用户、不同 topic 吸引用户的地理范围不同，和我们之前讨论的思路是非常契合的。我认为完全有理由将这种思路用在有多种商业类别的单一店铺上。
