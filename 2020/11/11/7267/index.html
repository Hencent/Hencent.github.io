DDTCDR: Deep Dual Transfer Cross Domain Recommendation

Posted on 2020-11-11 Edited on 2021-01-16 In Transfer-Learning

我理解的有很大偏差，下面这篇知乎讲的比较合理
https://zhuanlan.zhihu.com/p/111350866

文章主要解决什么问题

跨域推荐中，现有方法没有考虑 user-item 的双向潜在关系；也没有显式的使用 user item 特征，而是只用了评分数据。

解决的思路是什么

  • 使用正交映射提取多个域上的用户偏好
  • Deep Dual Transfer

模型分块说明

  1. embedding
    使用 auto-encoder 对原始特征进行 embedding

  2. 正交映射
    基本假设是，如果两个用户在源域中具有相似的偏好，则他们的偏好在目标域中也应相似。这意味着在跨领域迁移学习中，我们需要保持用户之间的相似性。
    因此，使用潜在正交矩阵跨域传输信息。由于正交变换保留了向量的内积，因此它保留了跨不同潜在空间的用户嵌入之间的相似性。

  3. Deep Dual Transfer Learning
    使用两个组件来建模用户偏好：域内偏好和跨域偏好，前者捕获用户交互并预测目标域中的用户行为，后者利用源域中的用户行为。
    第二项表示通过潜在正交映射函数X捕获跨域异质性而获得的跨域用户偏好。

    rA′=(1−α)RSA(WuA,WiA)+αRSB(X∗WuA,WiA)rB′=(1−α)RSB(WuB,WiB)+αRSA(XT∗WuB,WiB)\begin{array}{c} r_{A}^{\prime}=(1-\alpha) R S_{A}\left(W_{u_{A}}, W_{i_{A}}\right)+\alpha R S_{B}\left(X * W_{u_{A}}, W_{i_{A}}\right) \\ r_{B}^{\prime}=(1-\alpha) R S_{B}\left(W_{u_{B}}, W_{i_{B}}\right)+\alpha R S_{A}\left(X^{T} * W_{u_{B}}, W_{i_{B}}\right) \end{array}rA′​=(1−α)RSA​(WuA​​,WiA​​)+αRSB​(X∗WuA​​,WiA​​)rB′​=(1−α)RSB​(WuB​​,WiB​​)+αRSA​(XT∗WuB​​,WiB​​)​

其他

最原始的矩阵分解，可能只使用评分矩阵来分解，直接分解出 user 矩阵和 item 矩阵。本文使用了原始特征，训练 AE 参数，可能更合理一点，毕竟使用了不少原始特征。

NCF 倒没有什么特别的。

主要这个所谓的正交变换，作者说可以 由于正交变换保留了向量的内积，因此它保留了跨不同潜在空间的用户嵌入之间的相似性。，我暂时无法完全理解。

根据知乎,正交矩阵是保持两点的欧式距离不变的线性变换。也就是说：左乘正交矩阵造成的空间变换是用一个新空间代替原有空间，即用另一组正交基来描述被变换的向量，且不改变向量的长度和空间位置。比如固定向量不动，旋转原坐标系得到新的坐标系，并用这个新坐标系表述原向量。
还有另一个知乎

那么就可以理解为，经过这个转换，把 source 的 user 偏好，转换到了 target 的 item 相同的语义空间。因为假定的是：源空间和目标空间是有差异的，user的特征不能直接拿来用。

我准备把这个思想用在我们的建模中，不同城市中 品牌/类别 可以看作是 user，在不同城市中是不一样的。原文也写到，可以很方便的拓展到多领域（多城市）。
