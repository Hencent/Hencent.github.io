Spatiooral Dual Graph Attention Network for Query-POI Matching

Posted on 2020-10-23

文章主要解决什么问题

POI 的查询与匹配在各类 location-based 的服务中愈发重要。例如导航软件、打车软件。现在的应用场景需要根据用户输入的不完整的信息，来尽量准确的推荐 POI。

此外，除了应对普通的查询请求，更多的查询需要考虑不同的用户、不同的地点、不用的语境和时间关系，做出更精准的匹配。

文章可以应对三种困难：

  • 不确定性：同名 POI 可能有好几个地点
  • 复杂的当前状态: 例如需要考虑距离、当前时间等因素
  • 动态的用户兴趣: 用户的兴趣会发生变化，但和时间序列有关

解决的思路是什么

  • 使用一个语义提取模块，通过 CNN 等各种方法，提取搜索文本、POI名、时间、坐标、用户的特征信息，转为低维向量
  • 使用两个 Graph Attention Network，获取 POI 与 查询的相关性。第一个图用于获取通用的、相对地理位置有关的 query-POI 相关性，第二个图用于获取特定用户的、事件相关的 query-POI 相关性
  • 利用一个 fully-connected 网络来融合两个图获取到的信息
  • 利用 cos 或类似的相关性计算函数来获得 query-POI 的相关性

核心的知识点有哪些

  • Embedding
  • Graph Attention Network

模型分块说明

semantic representation

利用共享参数的 CNN 将 POI-name 和 query 编码为低维名称向量表示
按经纬度划分网格，将当前经纬度所在的网格和周围网格，使用 CNN 联合编码为当前位置对应的低维位置向量。再使用 CNN 来获取每一个查询对的低维向量表示。
将时间按一定间隔划分为时段。将当前时刻映射到对应时间段内，使用该时间段的向量表示
将每个用户的交互历史按照时段划分，将 item 按照对应的种类归类，得到用户在某个时间段内与某一类 item 交互的比例。然后使用 fully-connected 编码为低维的用户向量表示

dual graph attention network

generic query-POI graph

当某个 query 和某个 POI 之间的查询事件次数超过阈值，就连上一条权值为 1 的边。每个点的表示为上个模块中提取的语义信息. 使用 graph attention network 来获取普适的 query / POI 的 embedding。其中的 attention 为：

user-specific query-POI graph

此处以 POI 为例，对于 query 的处理类似。
首先，对于每个时段，都有不同的 user 发出的 quert-POI 查询信息图。由于之前提到了，前后时序的状态是有高度依赖关系的，所以可以推断，当前层状态是之前层状态的组合加上噪声。例如：

XP^tu,τ=∑i=1t1XP^t−iu,τZi+∑i=1t2εt−iu,τZi′\mathrm{X}_{\hat{P}_{t}^{u, \tau}}=\sum_{i=1}^{t_{1}} \mathrm{X}_{\hat{P}_{t-i}^{u, \tau}} Z_{i}+\sum_{i=1}^{t_{2}} \varepsilon_{t-i}^{u, \tau} Z_{i}^{\prime}XP^tu,τ​​=i=1∑t1​​XP^t−iu,τ​​Zi​+i=1∑t2​​εt−iu,τ​Zi′​

为了提高可解释性和时序依赖性，可以根据以下方法来扩展上式。
首先可以将噪声信息用语义表示、用户表示和时刻表示的组合：XP^tu,τ=Attn⁡s(u,τ,XPt)\mathrm{X}_{\hat{P}_{t}^{u, \tau}}=\operatorname{Attn}^{s}\left(u, \tau, \mathrm{X}_{P_{t}}\right)XP^tu,τ​​=Attns(u,τ,XPt​​)
其次，时序依赖关系可以表示为：Yt+1u,τ=∑k=0K1−1ψk(At)Ytu,τWk+XP^tu,τZ0\mathbf{Y}_{t+1}^{u, \tau}=\sum_{k=0}^{K_{1}-1} \psi_{k}\left(A_{t}\right) \mathbf{Y}_{t}^{u, \tau} W_{k}+\mathbf{X}_{\hat{P}_{t}^{u, \tau}} Z_{0}Yt+1u,τ​=∑k=0K1​−1​ψk​(At​)Ytu,τ​Wk​+XP^tu,τ​​Z0​，其中利用邻接矩阵等考虑了 K 跳邻居信息。总之，Y是一个隐层表示。
那么某时刻的输出最终可以表示为：Yt+1u,τ=∑k=0K1−1ψk(At)Ytu,τWk+XP^tu,τZ0\mathbf{Y}_{t+1}^{u, \tau}=\sum_{k=0}^{K_{1}-1} \psi_{k}\left(A_{t}\right) \mathbf{Y}_{t}^{u, \tau} W_{k}+\mathbf{X}_{\hat{P}_{t}^{u, \tau}} Z_{0}Yt+1u,τ​=∑k=0K1​−1​ψk​(At​)Ytu,τ​Wk​+XP^tu,τ​​Z0​

得到了之前每一个时段的 query / POI 表示之后，通过对过往信息的卷积操作，最终得到了当前时刻输出的 embedding. Xp^u,τ=Conv⁡(XP^t−tru,τ,XP^t−tr+1u,τ,…,XP^t−1u,τ)\mathrm{X}_{\hat{p} u, \tau}=\operatorname{Conv}\left(\mathrm{X}_{\hat{P}_{t-t r}^{u, \tau}}, \mathrm{X}_{\hat{P}_{t-t r+1}^{u, \tau}}, \ldots, \mathrm{X}_{\hat{P}_{t-1}^{u, \tau}}\right)Xp^​u,τ​=Conv(XP^t−tru,τ​​,XP^t−tr+1u,τ​​,…,XP^t−1u,τ​​)

总之，虽然有一些我暂时不能理解的卷积和 GCN 操作，但是大体的感觉是，通过邻居的 embedding 来丰富自身表示；利用上一个时刻的信息来影响下一时刻的信息。

pairwise neural fusion and ranking & output

将两个 graph 得到的 embedding 做一个 concatenation 操作拼接到一起，放入一个 fully-connected network 中得到 query 和 POI 最终的 embedding 表示。
最后使用 cos 相似度计算方法来计算相关性。

存在的问题 & 有哪些改进的思路

我个人觉得，把 POI 的 embedding 和 query 的 embedding 训练的一样并不太 make sense. 可能还是像 TransR / TransE 那样或许合理一点。那么时间信息和用户信息可以用来增强 edge 的信息。
