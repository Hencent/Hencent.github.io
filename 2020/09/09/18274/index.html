KGAT: Knowledge Graph Attention Network for Recommendation

Posted on 2020-09-09 In KnowledgeGraph

文章主要解决什么问题

利用知识图谱进行推荐

解决的思路是什么

使用知识图谱来构建 latent vector 进行推荐。

在 embedding 的基础上，利用临近节点的信息来迭代地更新 embedding vector，以获取几跳之外的关系，提高推荐准确率。

核心的知识点有哪些

  • attentive embedding propagation

模型分块说明

  1. embedding layer
    使用 TransR 和 pairwise ranking loss 来做 embedding

    g(h,r,t)=∥Wreh+er−Wret∥22g(h, r, t)=\left\|\mathbf{W}_{r} \mathbf{e}_{h}+\mathbf{e}_{r}-\mathbf{W}_{r} \mathbf{e}_{t}\right\|_{2}^{2}g(h,r,t)=∥Wr​eh​+er​−Wr​et​∥22​

    LKG=∑(h,r,t,t′)∈T−ln⁡σ(g(h,r,t′)−g(h,r,t))\mathcal{L}_{\mathrm{KG}}=\sum_{\left(h, r, t, t^{\prime}\right) \in \mathcal{T}}-\ln \sigma\left(g\left(h, r, t^{\prime}\right)-g(h, r, t)\right)LKG​=(h,r,t,t′)∈T∑​−lnσ(g(h,r,t′)−g(h,r,t))

  2. Attentive Embedding Propagation Layers
    这一模块及下一模块的组合是最关键的。用来收集与整合 K 跳内的 entity 信息，并赋予一定的权重。
    • 在每一层中，一个节点所获取到的邻域信息为 eNh=∑(h,r,t)∈Nhπ(h,r,t)et\mathbf{e}_{\mathcal{N}_{h}}=\sum_{(h, r, t) \in \mathcal{N}_{h}} \pi(h, r, t) \mathbf{e}_{t}eNh​​=∑(h,r,t)∈Nh​​π(h,r,t)et​
    • 每个三元组对该节点所施加的影响的权重是根据 embedding 在 relation r 的距离所决定的
      π(h,r,t)=(Wret)⊤tanh⁡((Wreh+er))\pi(h, r, t)=\left(\mathbf{W}_{r} \mathbf{e}_{t}\right)^{\top} \tanh \left(\left(\mathbf{W}_{r} \mathbf{e}_{h}+\mathbf{e}_{r}\right)\right)π(h,r,t)=(Wr​et​)⊤tanh((Wr​eh​+er​))
      π(h,r,t)=exp⁡(π(h,r,t))∑(h,r′,t′)∈Nhexp⁡(π(h,r′,t′))\pi(h, r, t)=\frac{\exp (\pi(h, r, t))}{\sum_{\left(h, r^{\prime}, t^{\prime}\right) \in \mathcal{N}_{h}} \exp \left(\pi\left(h, r^{\prime}, t^{\prime}\right)\right)}π(h,r,t)=∑(h,r′,t′)∈Nh​​exp(π(h,r′,t′))exp(π(h,r,t))​
  3. Information Aggregation
    将该节点的领域信息整合进该节点的 embedding

    fBi−Interaction=LeakyReLU(W1(eh+eNh))+ LeakyReLU (W2(eh⊙eNh))\begin{aligned} f_{\mathrm{Bi}-\mathrm{Interaction}}=& \mathrm{LeakyReLU}\left(\mathrm{W}_{1}\left(\mathrm{e}_{h}+\mathrm{e}_{N_{h}}\right)\right)+\\ & \text { LeakyReLU }\left(\mathrm{W}_{2}\left(\mathrm{e}_{h} \odot \mathrm{e}_{N_{h}}\right)\right) \end{aligned}fBi−Interaction​=​LeakyReLU(W1​(eh​+eNh​​))+ LeakyReLU (W2​(eh​⊙eNh​​))​

  4. 堆叠上述操作。堆叠的次数 L 实际上就是所能联系到的最大跳数。堆叠的更多就可以获取更深层的联系

    eh(l)=f(eh(l−1),eNh(l−1))\mathbf{e}_{h}^{(l)}=f\left(\mathbf{e}_{h}^{(l-1)}, \mathbf{e}_{\mathcal{N}_{h}}^{(l-1)}\right)eh(l)​=f(eh(l−1)​,eNh​(l−1)​)

    eNh(l−1)=∑(h,r,t)∈Nhπ(h,r,t)et(l−1)\mathbf{e}_{\mathcal{N}_{h}}^{(l-1)}=\sum_{(h, r, t) \in \mathcal{N}_{h}} \pi(h, r, t) \mathbf{e}_{t}^{(l-1)}eNh​(l−1)​=(h,r,t)∈Nh​∑​π(h,r,t)et(l−1)​

  5. Model Prediction
    将各次堆叠操作获取的不同 vector 拼接起来获取最终的表示。并用向量积表示匹配分数。

    eu∗=eu(0)∥⋯∥eu(L),ei∗=ei(0)∥⋯∥ei(L)\mathbf{e}_{u}^{*}=\mathbf{e}_{u}^{(0)}\|\cdots\| \mathbf{e}_{u}^{(L)}, \quad \mathbf{e}_{i}^{*}=\mathbf{e}_{i}^{(0)}\|\cdots\| \mathbf{e}_{i}^{(L)}eu∗​=eu(0)​∥⋯∥eu(L)​,ei∗​=ei(0)​∥⋯∥ei(L)​

    y^(u,i)=eu∗⊤ei∗\hat{y}(u, i)=\mathbf{e}_{u}^{* \top} \mathbf{e}_{i}^{*}y^​(u,i)=eu∗⊤​ei∗​
